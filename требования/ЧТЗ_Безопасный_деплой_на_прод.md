# ЧТЗ: Конвейер безопасного деплоя на прод (Blue-Green)

**ID:** ЧТЗ-001
**Дата:** 2026-05-20
**Автор:** Аналитик (AI Team Pipeline)
**Статус:** На утверждении

---

## 1. Описание

Создание безопасного CI/CD конвейера для деплоя DryClean Pro на production VPS с использованием стратегии **Blue-Green** через Docker Compose. Конвейер обеспечивает нулевой даунтайм, автоматический откат и полную проверку кода перед деплоем.

---

## 2. Проблемы текущего деплоя

| Проблема | Риск |
|----------|------|
| Нет CI-проверок перед деплоем | Сломанный код попадает на прод |
| Нет backup БД | Потеря данных при ошибке миграции |
| Нет health-check верификации | Деплой считается успешным, но сервисы не работают |
| Нет rollback-механизма | Ручное восстановление занимает часы |
| `StrictHostKeyChecking=accept-new` | MITM-атака |
| Пароль в sshpass | Менее безопасно чем SSH-ключи |
| Нет валидации env-переменных | Сервисы запускаются с дефолтными значениями |

---

## 3. Цели

1. **CI Pipeline:** Автоматические проверки (lint + type check) при push в main
2. **Blue-Green деплой:** Два окружения, переключение трафика через nginx без даунтайма
3. **Backup БД:** Автоматический backup перед каждым деплоем
4. **Health-check верификация:** Проверка всех сервисов после деплоя
5. **Автоматический rollback:** Откат на предыдущую версию при ошибке
6. **Безопасность:** SSH-ключи вместо паролей, валидация env
7. **Версионирование:** Семантическое версионирование образов + тегирование релизов

---

## 3.1 Версионирование — детали

### 3.1.1 Формат версии

Используется **SemVer 2.0.0**: `MAJOR.MINOR.PATCH`

| Компонент | Когда увеличивается | Пример |
|-----------|---------------------|--------|
| MAJOR | Breaking changes в API/БД | 2.0.0 |
| MINOR | Новые фичи (backward compatible) | 1.3.0 |
| PATCH | Баг-фиксы | 1.2.4 |

### 3.1.2 Автоматическое определение версии

| Тип коммита | Изменение версии | Conventional Commits |
|-------------|------------------|----------------------|
| `fix:` | PATCH +1 | `fix: исправлен баг авторизации` |
| `feat:` | MINOR +1 | `feat: добавлен личный кабинет` |
| `feat!:` / `BREAKING CHANGE:` | MAJOR +1 | `feat!: изменён формат API` |

### 3.1.3 Тегирование Docker образов

Каждый сервис получает теги:
```
dryclean/frontend:1.2.4          # конкретная версия
dryclean/frontend:1.2            # minor alias
dryclean/frontend:1              # major alias
dryclean/frontend:latest         # всегда последний стабильный
dryclean/frontend:blue           # текущий blue
dryclean/frontend:green          # текущий green
```

### 3.1.4 Реестр версий

Файл `VERSIONS.md` на сервере + git tag:
```
v1.2.4 | 2026-05-20 | fix: исправлен баг корзины | @author | DEPLOYED
v1.2.3 | 2026-05-19 | feat: добавлен фильтр        | @author | ROLLED_BACK
v1.2.2 | 2026-05-18 | fix: исправлен CSS           | @author | DEPLOYED
```

### 3.1.5 Хранение версий на сервере

```
/opt/app/
├── versions/
│   ├── v1.2.4/          # полный код версии
│   ├── v1.2.3/          # предыдущая (для rollback)
│   └── v1.2.2/          # предыдущая
├── backups/
│   ├── db_v1.2.4.sql    # backup БД перед v1.2.5
│   └── db_v1.2.3.sql
├── VERSIONS.md          # лог всех версий
└── current_version      # symlink → активная версия
```

### 3.1.6 Политика хранения

| Что | Сколько храним |
|-----|----------------|
| Docker образы | Последние 5 версий |
| Код на сервере | Последние 3 версии |
| Backup БД | Последние 7 версий |
| Git tags | Все (навсегда) |

| # | Критерий | Проверка |

| AC-08 | Health-check всех сервисов после деплоя | Script verification |
| AC-09 | Автоматический rollback при failed health-check | Rollback script |
| AC-10 | SSH-ключи вместо паролей | GitHub Secrets |
| AC-11 | Валидация env-переменных перед деплоем | Env check script |
| AC-12 | Логирование всех этапов деплоя | Deploy log output |
| AC-13 | Автоматическое определение версии по коммитам | SemVer parsing |

## 7. Архитектура Blue-Green

```
                    ┌─────────────┐
                    │   Nginx     │
                    │  :80/:443   │
                    └──────┬──────┘
                           │
              ┌────────────┴────────────┐
              │                         │
    ┌─────────▼─────────┐   ┌──────────▼─────────┐
    │   BLUE (active)   │   │  GREEN (standby)   │
    │  frontend:3001    │   │  frontend:3002     │
    │  content:8011     │   │  content:8012      │
    │  postgres:5433    │   │  postgres:5434     │
    │  redis:6380       │   │  redis:6381        │
    │  rabbitmq:5673    │   │  rabbitmq:5674     │
    └───────────────────┘   └────────────────────┘
              │                         │
              └─────────┬───────────────┘
                        │
              ┌─────────▼─────────┐
              │   Shared Volumes  │
              │  postgres_data    │
              │  content_uploads  │
              └───────────────────┘
```

**Принцип работы:**
1. Active = blue, Standby = green
2. Деплой идёт на standby (green)
3. Health-check green
4. Nginx переключает upstream на green
5. Green становится active, blue становится standby
6. При ошибке → rollback на blue

| Мера | Реализация |
|------|------------|

## 10. Версионирование — workflow

```
Push в main
    │
    ├─ CI: lint + type check
    │
    ├─ version.sh: парсит коммиты → v1.2.4
    │
    ├─ Build: docker build -t dryclean/service:1.2.4
    │         docker build -t dryclean/service:latest
    │         docker build -t dryclean/service:green
    │
    ├─ Deploy на standby (green)
    │
    ├─ Health-check green
    │
    ├─ Nginx: переключить upstream → green
    │
    ├─ version-registry.sh:
    │     ├─ git tag v1.2.4
    │     ├─ update VERSIONS.md
    │     └─ symlink current_version → v1.2.4
    │
    ├─ cleanup.sh: удалить образы старше 5 версий
    │
    └─ ГОТОВО: v1.2.4 active (green)
```

## 11. Риски и митигация

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Нехватка RAM для двух окружений | Средняя | Проверка RAM перед деплоем, cleanup old images |
| Потеря данных при миграции | Низкая | Backup БД перед деплоем |
| Nginx не переключил трафик | Низкая | Health-check после переключения |
| SSH подключение упало | Низкая | Retry logic в скрипте |
| Конфликт версий (duplicate tag) | Низкая | Проверка существующего тега перед созданием |
| Нехватка диска от старых образов | Средняя | Автоматический cleanup (хранить 5 версий) |
| Ошибка в SemVer расчёте | Низкая | Ручное переопределение версии через env |
