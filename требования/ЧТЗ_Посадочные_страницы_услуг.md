# ЧТЗ: Посадочные страницы услуг (Landing Pages)

## 1. Описание задачи

Создать посадочную страницу (landing page) для каждой услуги химчистки. При клике на услугу на главной странице клиент переходит на детальную страницу с описанием услуги, преимуществами, процессом работы, блогом/статьями и CTA-блоком для заявки.

## 2. Маршрутизация

**Маршрут 1 (Стандартный):** Аналитик → Разработчик → Тестировщик → DevOps

**Обоснование:** Задача затрагивает только frontend (Next.js). Не требует изменений в БД, API, архитектуре или межсервисных контрактах.

**Исполнитель:** Разработчик

## 3. Текущее состояние

- Компонент `Services.tsx` уже содержит ссылки `/uslugi/${service.slug}` ✅
- `constants.ts` содержит 6 услуг с id, title, slug, description, image ✅
- Footer уже ссылается на `/uslugi/himchistka-...` ✅
- Backend Content Service API доступен ✅
- Страницы `/uslugi/[slug]` и `/uslugi` **не созданы** ❌

## 4. Критерии приёмки

### AC-1: Страница всех услуг `/uslugi`
- Отображает все 6 услуг в виде карточек-ссылок
- Каждая карточка ведёт на `/uslugi/[slug]`
- SEO metadata настроена

### AC-2: Посадочная страница услуги `/uslugi/[slug]`
- Hero-блок с названием услуги, описанием и CTA-кнопкой
- Подробное описание услуги (длинный текст)
- Преимущества услуги (5-6 пунктов с иконками)
- Процесс работы (шаги)
- Блок "Полезные статьи" / блог (placeholder, 2-3 статьи)
- Секция "Другие услуги" (остальные услуги без текущей)
- CTA-блок "Оставить заявку" (заглушка, форма будет позже)
- SEO metadata (title, description) для каждой услуги
- Навигация (TopBar + Navigation) и Footer как на главной

### AC-3: Навигация
- Главная страница → клик на "Подробнее" у услуги → переход на `/uslugi/[slug]` (уже работает)
- Главная страница → "Все 12 услуг" → `/uslugi`
- Footer → ссылки на услуги → `/uslugi/[slug]`

### AC-4: Мобильная адаптация
- Все страницы адаптивны (mobile-first)
- Корректное отображение на мобильных, планшетах, десктопе

### AC-5: 404 обработка
- При переходе на несуществующий slug — страница 404 (notFound())

## 7. Технические решения

- **Статические данные** — контент услуг хранится в `serviceData.ts` (не API), для мгновенной загрузки и SEO
- **generateStaticParams** — предгенерация страниц всех услуг при сборке
- **generateMetadata** — динамический SEO для каждой услуги
- **Компоненты** — используются существующие UI-компоненты (Container, Section, Button)
- **Стили** — Tailwind CSS 4, следуя существующим паттернам проекта
