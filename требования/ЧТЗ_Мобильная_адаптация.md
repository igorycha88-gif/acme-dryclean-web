# ЧТЗ: Мобильная адаптация сайта

## Описание задачи
Адаптировать все компоненты главной страницы сайта DryClean Pro для корректного отображения на мобильных устройствах (320px–768px). Все изменения — только CSS/Tailwind классы, без изменения бизнес-логики.

## Маршрутизация
- **Маршрут:** Стандартный (Аналитик → Разработчик → Тестировщик → DevOps)
- **Исполнитель:** Разработчик
- **Архитектор:** Не требуется (только CSS-изменения)

## Критерии приёмки

### AC-01: Container — адаптивные отступы
- На экранах < 640px: `px-4` вместо `px-6`
- На экранах >= 640px: `px-6` (текущее поведение)

### AC-02: BeforeAfter — вкладки не вылазят за экран
- Вкладки категорий переносятся на новую строку (`flex-wrap`)
- Уменьшенный padding кнопок на мобильных (`max-md:px-3 max-md:py-1.5 max-md:text-xs`)

### AC-03: Services — карточки на мобильных
- Высота изображений: `h-48` на мобильных (вместо `h-72`)
- Padding карточек: `p-4` на мобильных (вместо `p-6`)
- Текст описания корректно переносится

### AC-04: Navigation — анимация мобильного меню
- Мобильное меню появляется с плавной анимацией (slide-down или fade-in)
- Overlay закрывается плавно

### AC-05: Hero — статистика
- Бейджи (рейтинг, заказы, оплата) корректно переносятся на мобильных
- Достаточный gap между строками

### AC-06: TopBar — информация на мобильных
- Показывать телефон и Telegram на мобильных
- Скрыть schedule/email (уже сделано) — оставить как есть

### AC-07: CTAForm — select на мобильных
- Добавить стрелку-индикатор для select (background-image)
- Корректные отступы и высота на мобильных

### AC-08: Promo — padding на мобильных
- Padding карточек: `p-6` на мобильных (вместо `p-8`)
- Размер заголовка: меньше на мобильных

### AC-09: B2BSection — padding на мобильных
- Padding карточек: `p-4` на мобильных (вместо `p-6`)

### AC-10: WhyUs — padding на мобильных
- Padding карточек: `p-4` на мобильных (вместо `p-6`)

### AC-11: Footer — планшетная раскладка
- На планшетах (md): 2x2 сетка вместо 1 колонки
- На мобильных (< md): 1 колонка (текущее поведение)
