# ЧТЗ: Интеграция мониторинга (Tracking + Prometheus + Grafana) в продакшн-деплой

**Дата:** 2026-06-01
**Приоритет:** CRITICAL
**Маршрут:** Инфраструктурная задача → DevOps

---

## 1. Описание проблемы

Пользователь заходит на da-dryclean.ru, кликает на услугу, но в Grafana нет данных. 

**Корневые причины:**
1. Tracking-сервис не запущен на проде (`deploy-vps.sh` не деплоит его)
2. Nginx не проксирует tracking-запросы (`nginx/prod.conf` нет location)
3. Prometheus и Grafana не подключены к прод-сети (в `docker-compose.monitoring.yml` изолированная сеть `172.30.0.0/16`)
4. Frontend отправляет события на `http://localhost:8020` (fallback в `tracker.ts`), что не работает из браузера
5. CI/CD не собирает tracking-образ (`deploy.yml` только frontend + content)

## 2. Решение

Встроить Tracking + Prometheus + Grafana в продакшн-деплой (`deploy-vps.sh`), единую сеть `dryclean-net`, все контейнеры связаны по именам.

**Архитектура прод-сети после фикса:**
```
dryclean-net (bridge):
  dryclean-postgres    → port 5432 (internal only)
  dryclean-content     → port 8011 (host-mapped)
  dryclean-frontend    → port 3000 (host-mapped)
  dryclean-tracking    → port 8020 (host-mapped) [NEW]
  dryclean-prometheus  → port 9090 (host-mapped) [NEW]
  dryclean-grafana     → port 3030:3000 (host-mapped) [NEW]

Host nginx (80/443):
  / → 127.0.0.1:3000 (frontend)
  /api/ → 127.0.0.1:8011 (content)
  /api/v1/tracking/ → 127.0.0.1:8020 (tracking) [NEW]

Grafana: IP:3030 (прямой доступ, пароль admin)
```

## 3. Критерии приёмки

| AC# | Критерий | Проверка |
|-----|----------|----------|
| AC-1 | Frontend отправляет tracking-события через nginx (относительный URL `/api/v1/tracking/event`) | Браузер → DevTools → Network → POST `/api/v1/tracking/event` → 200 |
| AC-2 | Nginx проксирует `/api/v1/tracking/` → tracking-контейнер | `curl -X POST https://da-dryclean.ru/api/v1/tracking/event` → не 404 |
| AC-3 | Tracking-контейнер запущен и healthy на проде | `docker ps` → `dryclean-tracking` → `healthy` |
| AC-4 | Таблицы analytics_events и analytics_sessions созданы | `docker exec dryclean-postgres psql -U dryclean -c '\dt analytics*'` |
| AC-5 | Prometheus собирает метрики с tracking и content | `curl http://VPS_IP:9090/api/v1/targets` → все targets UP |
| AC-6 | Grafana доступна на IP:3030 | `curl -f http://VPS_IP:3030/api/health` → 200 |
| AC-7 | Grafana подключена к PostgreSQL и Prometheus | Dashboard "D&A Dry Cleaning — Бизнес-аналитика" показывает данные |
| AC-8 | Клик на услугу → событие в БД → отображается в Grafana | E2E проверка |
| AC-9 | CI/CD собирает tracking-образ | GitHub Actions → Docker Build → tracking image pushed |
| AC-10 | deploy-vps.sh деплоит все 6 контейнеров (postgres, content, frontend, tracking, prometheus, grafana) | Скрипт завершается без ошибок |
