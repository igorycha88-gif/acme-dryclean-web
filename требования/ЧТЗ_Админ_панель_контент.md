# ЧТЗ: Админ-панель управления контентом

## 1. Общее описание

| Параметр | Значение |
|----------|----------|
| **Название** | Content Management Admin |
| **Тип** | Веб-приложение (админ-панель) |
| **Суть** | Панель управления контентом сайта: услуги, FAQ, отзывы, медиафайлы |
| **Целевые пользователи** | Администраторы, контент-менеджеры |

---

## 2. Функциональные требования

### 2.1 Разделы админ-панели

| № | Раздел | Описание |
|---|--------|----------|
| 1 | **Дашборд** | Обзор статистики (кол-во услуг, отзывов, FAQ) |
| 2 | **Услуги** | CRUD операций с услугами |
| 3 | **FAQ** | Управление вопросами-ответами |
| 4 | **Отзывы** | Управление отзывами клиентов |
| 5 | **Медиа** | Загрузка и управление изображениями |
| 6 | **Страницы** | Редактирование контента секций страниц |

### 2.2 Сущности и поля

#### 2.2.1 Услуги (services)

| id | UUID | Да | Уникальный идентификатор |

| description | text | Да | Описание услуги |
| image_url | string(500) | Нет | URL изображения |

| category | string(100) | Нет | Категория |
| is_active | boolean | Да | Активность (по умолч. true) |
| sort_order | integer | Да | Порядок сортировки |
| created_at | datetime | Да | Дата создания |
| updated_at | datetime | Да | Дата обновления |

#### 2.2.2 FAQ (faq)

| Поле | Тип | Обяз. | Описание |
|------|-----|-------|----------|
| id | UUID | Да | Уникальный идентификатор |

| is_active | boolean | Да | Активность |
| sort_order | integer | Да | Порядок сортировки |

#### 2.2.3 Отзывы (reviews)

### 2.3 API эндпоинты

#### Services
```
GET    /api/v1/content/services          - Список услуг (пагинация, фильтры)
POST   /api/v1/content/services          - Создать услугу
GET    /api/v1/content/services/{id}     - Детали услуги
PATCH  /api/v1/content/services/{id}     - Обновить услугу
DELETE /api/v1/content/services/{id}     - Удалить услугу
POST   /api/v1/content/services/reorder  - Изменить порядок
```

#### FAQ
```
GET    /api/v1/content/faq               - Список FAQ
POST   /api/v1/content/faq               - Создать FAQ
GET    /api/v1/content/faq/{id}          - Детали FAQ
PATCH  /api/v1/content/faq/{id}          - Обновить FAQ
DELETE /api/v1/content/faq/{id}          - Удалить FAQ
POST   /api/v1/content/faq/reorder       - Изменить порядок
```

#### Reviews
```
GET    /api/v1/content/reviews           - Список отзывов
POST   /api/v1/content/reviews           - Создать отзыв
GET    /api/v1/content/reviews/{id}      - Детали отзыва
PATCH  /api/v1/content/reviews/{id}      - Обновить отзыв
DELETE /api/v1/content/reviews/{id}      - Удалить отзыв
PATCH  /api/v1/content/reviews/{id}/toggle - Переключить активность
```

#### Media
```
GET    /api/v1/content/media             - Список медиафайлов
POST   /api/v1/content/media/upload      - Загрузить файл
GET    /api/v1/content/media/{id}        - Детали файла
DELETE /api/v1/content/media/{id}        - Удалить файл
```

### 2.4 UI/UX требования

| Требование | Описание |
|------------|----------|
| **Адаптивность** | Полная поддержка desktop (1024px+) |
| **Навигация** | Боковая панель (sidebar) с разделами |

## 4. Критерии приёмки

### 4.1 Успешная реализация

- [ ] Страница `/admin` доступна и показывает дашборд
- [ ] Страница `/admin/services` — таблица услуг с пагинацией
- [ ] Создание/редактирование/удаление услуг работает
- [ ] Страница `/admin/faq` — CRUD для FAQ
- [ ] Страница `/admin/reviews` — CRUD для отзывов
- [ ] Страница `/admin/media` — загрузка изображений
- [ ] Загруженные фото отображаются в списке и могут быть скопированы URL
- [ ] Все формы имеют валидацию
- [ ] Toast-уведомления работают

### 4.2 Визуальная проверка

- [ ] Sidebar отображается слева
- [ ] Активный раздел подсвечен
- [ ] Таблицы имеют пагинацию
- [ ] Формы имеют лейблы и плейсхолдеры
- [ ] Кнопки действий видны (редактировать, удалить)

## 5. Техническая реализация

### 5.1 Структура проекта

```
/backend/services/content/           # Content Service (FastAPI)
  /app/
    /api/v1/content/
      services.py                    # CRUD endpoints
      faq.py
      reviews.py
      media.py
    /models/                         # SQLAlchemy models
    /schemas/                        # Pydantic schemas
    /services/                       # Business logic
  alembic/                           # Миграции
  Dockerfile
  requirements.txt

/frontend/
  /src/app/admin/                    # Админ-панель
    /layout.tsx                      # Admin layout с sidebar
    /page.tsx                        # Дашборд
    /services/
      page.tsx                       # Список услуг
      new/page.tsx                   # Создание
      [id]/page.tsx                  # Редактирование
    /faq/
    /reviews/
    /media/
  /src/lib/api.ts                    # API клиент
```

### 5.2 Зависимости

**Backend:**
- FastAPI
- SQLAlchemy 2.0 (async)
- Pydantic v2
- aio-files (загрузка файлов)
- python-multipart

**Frontend:**
- Next.js App Router
- shadcn/ui (уже используется)
- react-hook-form + zod
- axios
